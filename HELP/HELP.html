Splitting File

Drag files from Windows Explorer and simply drop on the button on left top corner.

By default split files are directed to the directory of the Input file. Note that the Input File is not affected. The program also writes batch file which can be used merge split files. Hence this application is not required to merge the split files.

Choosing Split Size

Two Options are provided to choose size of batch file

1) Predefined:

This consists of predefined batch sizes like 1MB, Floppy Disc(1.38 MB), CD(650 MB) etc.. The drop down box allows to choose any of the choices in it.

2) Manual:

If you are not satisfied with predefined sizes you can choose custom size by selecting "Manual" RadioButton. A TextField is provided to Enter the custom Size followed by a drop down box having desired scaling factors.

Splitting Manually

If you desire to split manually, check the "Manual Split" Box in the area titled "Split Manually". This enables a browse button. Click on the "Browse" button to select the Input File. When done the button text changes to "Split". Now click on the button named "Split" to split.

Note: If you don't choose any file the Button text remains as "Browse".

  back to top

*************************************************************

 

Clear Manual Split

Suppose if you choose a wrong Input File in the Manual Selection and you don't want to split it OR if you decide choose another file, then in order to clear the old file simply uncheck the "Clear Selection" box which clears the file to be split.

Note: While checking "Manual Split" its text changes to "Clear Selection".

Redirect Split Files

If you want to redirect the output split files to another directory just check on "ReDirect Output" box found in  Options area. This enables a "Browse" button adjoining to it. Clicking on this pops up a Output Directory Chooser Dialog. Choose the desired Output Directory from the dialog.

Command Line Options:

To use command line option after setting classpath follow the syntax as given below:

java -jar FileSplitter.jar <source_file> <Split Size> <Destination_Path>
 

The last argument is optional. If Nothing specified in last argument the default directory in which the source file resides is chosen as Destination. Note: you need to type full file paths for Source as well as Destination.

Examples:

example 1: java -jar FileSplitter.jar <drive>:\My_Folder\Myfile 12345

example 2: java -jar FileSplitter.jar <drive>\My_Folder\Myfile 12345 <Drive>:\MyOutputPath

Checking File Values:

Java provides various methods for checking the security of a stream. I have simply implemented those streams. This includes CRC32,MD5,MD2,SHA. Why is this included here?

1) For my Software.

2) General Purpose.

Before Splitting the software computes either of values specified above. Although not implemented yet, the future versions would definitely include this feature. One of the implication is to check original file value against  merged file.

Also it can be used for checking any files. Remember for huge files say above 200MB, the computation is slower and the program appears to be hanged. But this is not the case. Please wait for a while (definitely not more than 1 minute up to file size of 800MB ). Don't forget to refresh your OS after computing for large files.

Open Destination Folder:

In the options area just check/uncheck "Open Destination Folder" to enable/disable the feature. This feature enables you to open the destination folder where the split files are directed. This feature is implemented by  integrating with Windows Explorer.
